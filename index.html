Important Concepts

Section 1: Web basics and HTML basics

The computer and the Web

One can tell a computer what to do by writing text that it can understand. The web is a bunch of computers that communicate with each other. When a user goes to a web page, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video explains completely.

HTML

HTML stands for Hypertext Markup Language.
HTML is made up of:
- Text: written content
- Markup or what the text looks like; (markup is used to make it look different)
- May have references to other files, say images or videos
- May have links to other pages

This video gives a good basic overview of HTML.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags.
HTML tags are shown in angled brackets. Most consist of an opening tag and a closing tag. Some do not have a closing tag and are called void tags.
Everything within a set of opening and closing tags is an element. This video explains this in detail, including the distinction between these and the concept of an attribute as well.

Need to note about Computers

Computers interpret instructions literally, and need to be told everything clearly and exactly. Programmers need to write exactly the way a computer understands, that is they need to write with correct syntax. A small mistake by a programmer can cause huge problems in a computer program.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them. More on the box structure in the next section.

Section 2: Creating a Structured Document with HTML

Visualising concepts of 'Structure' and 'Style'

Why Front-end web development is good to start programming with because:
- Its fun
- Its inherently visual: result immediately visible in browser
- One can get immediate feedback using it, and can share work with others

HTML and CSS are both languages. If one thinks of a website as a house, HTML defines the structure of the house. CSS defines the style of the house. Javascript adds interactive components changing some elements of the house. It can be thought to make things within this house move or be automated.
Each of these have rules that define how web-sites behave. In addition, various tools can be used as well to help build a website, similar to building a house.

To recap:
HTML: HyperText Markup Language - the standard markup language used to create web pages.
CSS: Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

Tree-like Structure of HTML

When we see the structure of a web page:
- Not all observable elements in the structure are visible on the page
- Every element takes up a rectangular space on page
- By going deep enough into the page structure, one can find all the text visible on the page

This structure of a web page is referred to as a Tree-like structure. The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. This relationship can be drawn like a family tree. In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on. When programmers talk about the DOM (document object model), they are talking of the tree-like structure of the web page.

DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages).
The nodes of every document are organized in a tree structure, called the DOM tree.

HTML is a language with syntax and rules. The basic compoments of HTML, the tags make up the elements in the tree, which the browser reads from the DOM to display on the webpage.

Boxes and Indentation

Container elements (span and div) are used to group related content together on a web page. Everything on a web page is actually a rectangular BOX. These boxes can be considered to be arranged on an underlying grid on the page. Boxes can be defined using the div tag. They can have labels to specify location on the page; can also have class names to define purpose.

HTML structure is indented to make it easy to read. When we read an HTML document as text, we see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested it's corresponding "box" is.

A Note on Programming Text Editors

When writing code, programmers use special text editors (for instance, Sublime Text). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag. Technically, a plain-text editor such as Notepad can also be used, as long as the file name ends in .html. But Notepad is typically meant for writing documents for reading, and is not designed for writing computer code. It does not have the features and ease of use provided by a programming text editor.

Section 3: Adding CSS Style to HTML Structure

How to include CSS Styling in a web page

CSS enables styling, such as various font sizes and border widths, on the web page. CSS is code written to control the style of HTML elements. There can be several CSS sheets linked to an html file. The HTML file tells the browser where to find the related style to apply - the link to the CSS style file is included in the head tag of the html.

There are three ways to include CSS styling in a webpage:

Method 1: write CSS in the head of your HTML. This method is good for very small projects. At the top of your HTML document, add a head element. Inside the head element, add a style element. Write your CSS in the style element and then put the HTML for the rest of your page below.

Method 2: Link your HTML to a separate CSS file. This adds another step, but it lets you stay more organized when working on larger projects.
Write all of your structural HTML in one file (lets call it main.html). In a separate file (let's call it main.css), put all of your CSS code. Add a head element to the top of your HTML. Add a link tag inside the head element. Since link is a "void tag" you don't need to add a closing link. Add the following attributes to your link: * rel="stylesheet" * type="text/css" * href="main.css"

Method 3: Write your style inline with your HTML. This is generally a very bad idea because it leads to lots of repeated code. You basically modify the style attributes of every individual HTML element.

Avoiding Repetition by using CSS

To recall, CSS stands for "Cascading Style Sheets". Cascading refers to going down the style code till the specific rule for the element is found. The most specific rules defined for a particular element are the ones that apply.

Inheritance is another key feature in CSS. It relies on the ancestor-descendant relationship in the page structure, to operate. Inheritance is the mechanism by which properties are applied not only to a specified element, but also to its descendants. Inheritance relies on the document tree of HTML elements in a page. Descendant elements may inherit CSS property values from any ancestor element enclosing them.
In general, descendant elements inherit text-related properties, but box-related properties are not inherited.

Cascading and Inheritance prevent certain properties from being declared over and over again in a style sheet, allowing the software developers to write less CSS. This enhances faster-loading of web pages by users and enables the clients to save money on bandwidth and development costs. Avoiding Repetition is a very useful benefit of using CSS effectively.

Avoiding repetition is important also for:

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere that the repeated code appears, problems will arise.

Consistency: In order for each section in a web page to look the same, manually giving each one the same style is not efficient or error-proof. Instead, each section div can be given the same class and then the style for that class can be specified in only one place. By avoiding repetition, it's now very easy to make changes to this standard style.

To sum, the advantages for having a different css file are:
- Easy to maintain your html page
- Change to the Look and feel will be easy and you can have support for many themes on your pages
- Your css file will be cached on the browser side. So you will contribute a little on internet traffic by not loading some kbs of data every time the page is refreshed or user navigates your site.
